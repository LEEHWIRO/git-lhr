* 시나리오
 1. 이벤트 목록 조회
 1-1 전체 이벤트 목록을 출력한다. 
 - 출력 컬럼 [이벤트 번호 SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜]
 1-2. 현재 로그인 상태여부를 판별한다.[고객,가맹점인지 관리자인지]
 1-3. 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 등록  2. 전체 이벤트 목록  3. 현재 이벤트 목록  4. 종료된 이벤트 목록  5. 상세조회  0. 이전으로]
 1-4. 관리자는 1. 이벤트 등록를 선택한다. 
 1-5. 관리자는 이벤트 등록에 필요한 정보를 입력한다.
 - 입력내용 : [이벤트 제목, 이벤트 시작날짜(YYYY-MM-DD), 이벤트 종료날짜(YYYY-MM-DD), 이벤트 내용]
 1-6. 관리자는 이벤트 등록에 필요한 모든 정보를 입력했다.
 1-7. 전체 이벤트 목록을 출력한다.
 - 출력 컬럼 [이벤트 번호 SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜]
 1-8. 현재 로그인 상태여부를 판별한다.[고객,가맹점인지 관리자인지]
 1-9. 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 등록  2. 전체 이벤트 목록  3. 현재 이벤트 목록  4. 종료된 이벤트 목록  5. 상세조회  0. 이전으로]
 ** 전체 이벤트 목록, 현재 이벤트 목록, 종료된 이벤트 목록은 모든 사용자에게 존재하므로 'AllUserService'를 제작한다.
 1-10 관리자는 5. 상세조회 버튼을 클릭한다.
 1-11. 관리자는 상세 조회할 이벤트 게시글 번호를 입력한다.
 1-12. 관리자는 이벤트의 상세 정보를 확인한다.
 - 출력 컬럼 [이벤트 번호SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜, 이벤트 내용, 이벤트 작성일자(YYYY-MM-DD)]
 1-13. 관리자는 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 수정  2. 이벤트 삭제  0. 이전으로]
 1-14. 관리자는 1. 이벤트 수정 메뉴를 선택한다.
 1-15. 관리자는 이벤트 제목, 이벤트 내용, 이벤트 시작날짜, 이벤트 종료날짜를 입력한다. (아무것도 입력하지 않은 경우 이전 정보를 사용한다.)
 1-15-2. 관리자는 수정사항을 모두 입력했다.
 1-16(중간에 추가된 내용임). 전체 이벤트 목록을 출력한다.
 - 출력 컬럼 [이벤트 번호 SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜]
  
 1-16. 현재 로그인 상태여부를 판별한다.[고객,가맹점인지 관리자인지]
 1-17. 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 등록  2. 전체 이벤트 목록  3. 현재 이벤트 목록  4. 종료된 이벤트 목록  5. 상세조회  0. 이전으로]
 1-18. 관리자는 5. 상세조회 버튼을 클릭한다.
 1-19. 관리자는 상세 조회할 이벤트 게시글 번호를 입력한다.
 1-20. 관리자는 이벤트의 상세 정보를 확인한다.
 - 출력 컬럼 [이벤트 번호SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜, 이벤트 내용, 이벤트 작성일자(YYYY-MM-DD)]
 1-21. 관리자는 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 수정  2. 이벤트 삭제  0. 이전으로]
 1-22. 관리자는 2.이벤트 삭제 메뉴를 클릭한다.
 1-23. 관리자가 삭제하고자 하는 이벤트가 삭제된다.
 1-24(중간에 추가된 내용임). 전체 이벤트 목록을 출력한다.
 - 출력 컬럼 [이벤트 번호 SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜]
 1-24. 현재 로그인 상태여부를 판별한다.[고객,가맹점인지 관리자인지]
 1-25. 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 등록  2. 전체 이벤트 목록  3. 현재 이벤트 목록  4. 종료된 이벤트 목록  5. 상세조회  0. 이전으로]
 1-26. 관리자는 5. 상세조회 버튼을 클릭한다.
 1-27. 관리자는 상세 조회할 이벤트 게시글 번호를 입력한다.
 1-28. 관리자는 이벤트의 상세 정보를 확인한다.
 - 출력 컬럼 [이벤트 번호SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜, 이벤트 내용, 이벤트 작성일자(YYYY-MM-DD)]
 1-29. 관리자는 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 수정  2. 이벤트 삭제  0. 이전으로]
 1-30. 관리자는 0. 이전으로 버튼을 클릭한다.
 1-31(중간에 추가된 내용임). 전체 이벤트 목록을 출력한다.
 - 출력 컬럼 [이벤트 번호 SEQ, 이벤트 제목, 이벤트 시작날짜, 이벤트 종료날짜]
 1-24. 현재 로그인 상태여부를 판별한다.[고객,가맹점인지 관리자인지]
 1-25. 관리자 메뉴를 출력한다.
 - 출력 메뉴 [1. 이벤트 등록  2. 전체 이벤트 목록  3. 현재 이벤트 목록  4. 종료된 이벤트 목록  5. 상세조회  0. 이전으로]
 1-16. 관리자는 이벤트에 대한 정보를 모두 제공받았으며, 전체 이벤트 목록 현재 이벤트 목록, 종료된 이벤트 목록에 대한 정보는 고객과 동일시 한다.
 1-17. 관리자는 0.이전으로 버튼을 클릭한다.
 1-18. 소분류 메뉴로 이동한다.
 - 출력 메뉴 [1. 메뉴소개 2. 공지사항 3. 이벤트 4. 가맹관리 5. 고객센터 0. 로그아웃]
